## useRef에 대해 설명하시오.

useRef는 저장 공간 또는 DOM 요소에 접근하기 위해 사용되는 React Hook입니다. useRef를 사용하여 관리하는 값은 값이 변해도 화면이 다시 렌더링되지 않습니다.

React에서 state 값이 변경되면 해당 컴포넌트가 다시 렌더링됩니다. 이는 가상 DOM의 변경된 값을 수정하여 화면에 반영하는 과정입니다. 그러나 useRef를 사용하여 DOM 요소를 직접 수정하는 경우, 해당 변경은 React의 렌더링 메커니즘과는 독립적으로 처리됩니다. 따라서 useRef로 관리하는 값이 변경되어도 컴포넌트의 리렌더링이 발생하지 않습니다.

부가적인 설명으로, useRef는 주로 DOM 요소에 접근하기 위해 사용됩니다. useRef를 사용하여 DOM 요소에 접근하고 조작할 수 있으며, 이는 주로 애니메이션, 타이머, 외부 라이브러리와의 통합 등에 유용합니다. useRef로 관리되는 값은 컴포넌트의 생명주기 동안 유지되며, 값의 변경이 발생해도 컴포넌트의 재렌더링을 유발하지 않습니다.

한편, useRef로 관리되는 값이 변경되었을 때 컴포넌트를 강제로 리렌더링하고 싶다면, useRef의 .current 속성을 수정하는 방법을 사용할 수 있습니다. 이 경우 컴포넌트는 useRef를 참조하는 곳에서 다시 렌더링됩니다. 하지만 이러한 사용은 일반적으로 권장되지 않으며, 상태 관리를 위해 useState나 useReducer를 사용하는 것이 좋습니다.

## useEffect의 실행순서에 대해 설명하시오.

요약하면, useEffect는 컴포넌트의 초기 렌더링 시에 실행되고, 이후 컴포넌트의 업데이트 시에도 실행됩니다. 만약 의존성 배열을 제공한 경우, 의존성 배열에 포함된 값이 변경될 때에만 useEffect가 실행됩니다. 마지막으로, 컴포넌트가 언마운트되기 전에 클린업 함수를 반환하여 정리 작업을 수행할 수 있습니다.

## https란

HTTPS(HTTPS://)는 HyperText Transfer Protocol Secure의 약자로, 웹 브라우저와 웹 서버 간의 통신을 안전하게 암호화하는 프로토콜입니다. 기존의 HTTP 프로토콜과는 달리 데이터의 안전성과 보안을 강화하기 위해 사용됩니다.

HTTPS는 다음과 같은 특징을 가지고 있습니다:

데이터 암호화: HTTPS는 데이터를 암호화하여 전송합니다. 이를 위해 SSL(보안 소켓 계층) 또는 TLS(전송 계층 보안) 프로토콜을 사용합니다. 암호화된 데이터는 도청이나 데이터 변조를 방지할 수 있습니다.

인증과 신뢰성: HTTPS는 서버와 클라이언트 간의 상호 인증을 수행합니다. 웹 서버는 디지털 인증서를 사용하여 자신의 신원을 증명하고, 클라이언트는 이를 확인하여 서버의 신뢰성을 검증합니다.

데이터 무결성: HTTPS는 데이터 무결성을 보장합니다. 암호화와 해시 함수를 사용하여 전송 중에 데이터가 변경되지 않았는지 확인합니다. 이를 통해 데이터가 완전하고 변조되지 않았음을 보장할 수 있습니다.

보안 접속 설정: HTTPS는 웹 서버와 클라이언트 간의 안전한 접속을 설정합니다. 이를 위해 SSL/TLS 핸드셰이크 프로세스를 통해 암호화 키를 교환하고 보안 연결을 설정합니다.

HTTPS는 주로 개인 정보, 금융 거래, 로그인 정보 등을 포함한 민감한 데이터를 전송해야 할 때 사용됩니다. 또한, 검색 엔진 최적화(SEO) 측면에서도 HTTPS 사용이 권장되며, 최신 웹 브라우저에서는 HTTPS를 사용하지 않는 웹 사이트에 대해 경고를 표시하기도 합니다.
웹 사이트를 HTTPS로 전환하려면 SSL/TLS 인증서를 구매하고 설치해야 합니다.

## http랑 https의 차이점

보안성:

HTTP: HTTP는 암호화되지 않은 텍스트로 데이터를 전송합니다. 따라서, 데이터는 도청될 수 있으며, 중간에 있는 공격자가 데이터를 수정하거나 변조할 수 있습니다.
HTTPS: HTTPS는 데이터를 암호화하여 전송합니다. SSL(보안 소켓 계층) 또는 TLS(전송 계층 보안) 프로토콜을 사용하여 데이터의 기밀성을 보장하고, 데이터의 무결성을 검증하여 중간에 데이터가 변경되지 않았음을 보장합니다.
암호화 프로토콜:

HTTP: HTTP는 암호화를 지원하지 않습니다.
HTTPS: HTTPS는 SSL 또는 TLS 프로토콜을 사용하여 데이터를 암호화합니다. 이를 통해 데이터가 안전하게 전송됩니다.
포트 번호:

HTTP: HTTP는 기본적으로 80번 포트를 사용합니다.
HTTPS: HTTPS는 기본적으로 443번 포트를 사용합니다.
인증서:

HTTP: HTTP는 인증서를 사용하지 않습니다.
HTTPS: HTTPS는 웹 서버의 신원을 확인하기 위해 디지털 인증서를 사용합니다. 인증서는 인증 기관에 의해 발급되며, 클라이언트는 이를 확인하여 서버의 신뢰성을 검증합니다.

검색 엔진 최적화(SEO):

HTTP: HTTP는 검색 엔진 최적화(SEO) 측면에서 보안 요구 사항을 충족시키지 않습니다. 최신 웹 브라우저는 HTTPS를 사용하지 않는 웹 사이트를 보안 위험으로 간주하고 경고를 표시하기도 합니다.
HTTPS: HTTPS는 검색 엔진 최적화(SEO)에 긍정적인 영향을 미칩니다. HTTPS를 사용하는 웹 사이트는 안전하다고 인식되며, 검색 엔진에서 더 높은 순위를 받을 수 있습니다.

## tcp란 무엇인가

TCP (Transmission Control Protocol)는 인터넷 프로토콜 스위트의 핵심 프로토콜 중 하나로, 네트워크 상에서 데이터를 안정적으로 전송하기 위해 사용됩니다. TCP는 데이터의 신뢰성과 흐름 제어를 담당하며, 패킷 기반의 통신을 지원합니다.

신뢰성: TCP는 데이터의 신뢰성을 보장하기 위해 여러 기능을 제공합니다. 데이터를 세그먼트라는 작은 단위로 분할하여 전송하며, 수신 측은 세그먼트를 확인하고 손상된 데이터를 다시 요청합니다. 또한, 데이터 전송의 정확성을 검증하기 위해 확인 응답(ACK) 메커니즘을 사용합니다.

흐름 제어: TCP는 흐름 제어 기능을 통해 데이터의 양을 조절하고 네트워크의 혼잡을 방지합니다. 수신 측은 송신 측에게 자신의 버퍼 상태를 알려주고, 수용 가능한 데이터 양을 조정하여 과도한 데이터 전송을 방지합니다.

순서 보장: TCP는 데이터의 순서를 보장합니다. 전송된 데이터는 수신 측에서 도착한 순서대로 재조립되어 애플리케이션에 전달됩니다. 이를 위해 TCP는 각 세그먼트에 일련 번호를 할당하여 순서를 추적합니다.

연결 지향성: TCP는 연결 기반 프로토콜로, 통신 전에 수신 측과 송신 측 간에 연결을 설정해야 합니다. 이를 통해 신뢰성과 순서 보장을 달성하며, 데이터의 전송 및 수신에 대한 제어를 가능하게 합니다.

다중화 및 다중화: TCP는 단일 IP 주소에서 다중 포트를 사용하여 여러 응용 프로그램 간의 동시 통신을 지원합니다. 이를 통해 한 컴퓨터에서 여러 개의 TCP 연결을 생성하여 동시에 다른 호스트와 통신할 수 있습니다.
